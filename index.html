删除win7右键菜单intel的图形属性和图形选项

发表于 2019-09-19 分类于 windows


说明 ：如上图，电脑有intel集显的，装好驱动，一般会在桌面右键菜单添加图形属性和图形选项两项(如上图红色框内)，为什么要删除这两项，因为有这两个选项时，每点一次右键，资源管理器进程(explorer.exe)占用的内存都会增加6-8M的内存，最开始explorer.exe也就占用10M多点内存，我们在使用过程中，都会用到右键，或者还会不自觉的右键刷新，这样explorer.exe占用的内存就会越来多，这似乎是intel的一个小bug吧
把右键菜单这两项(图形属性和图形选项)去掉，就没有了这种现象

一般都只有下面两个选项，有些个别电脑可能会有4个选项，删除intel的那两项就可以了(下面两个前面带图标的，很明显)
方法：
打开注册表，找到以下目录：

上面两个选项
删除：运行中输入：regsvr32 /u igfxpph.dll
恢复：运行中输入：regsvr32 igfxpph.dll
下面两个选项
删除：运行中输入：regsvr32 /u igfxdtcm.dll
恢复：运行中输入：regsvr32 igfxdtcm.dll

释放linux的swap内存

发表于 2019-09-18 分类于 linux


如上图，一般情况下不会用到swap的，一般物理内存使用在90%以上(默认是这个数，可以自行在/etc/sysctl.conf里设置vm.swappiness参数)，但有的时候，内存会被缓存占用，导致系统开始使用swap空间，此时就需要清理下swap了
这里先说下vm.swappiness参数，设置成vm.swappiness＝10：就是说当内存使用90%以上才会使用swap空间

###第一步：先执行sync命令

sync命令用于强制被改变的内容立刻写入磁盘，更新超块信息，以防止释放，sync命令则可用来强制将内存缓冲区中的数据立即写入磁盘中。
第二步：(如果仅仅是清理swap的话，这一步可以不执行)

此指令输入立即生效，意在释放所有缓存。
关于drop_caches：
drop_caches的详细文档如下：
Writing to this will cause the kernel to drop clean caches, dentries and inodes from memory, causing that memory to become free.
To free pagecache:

  • echo 1 > /proc/sys/vm/drop_caches
    To free dentries and inodes:
  • echo 2 > /proc/sys/vm/drop_caches
    To free pagecache, dentries and inodes:
  • echo 3 > /proc/sys/vm/drop_caches
    As this is a non-destructive operation, and dirty objects are notfreeable, the user should run “sync” first in order to make sure allcached objects are freed.
    This tunable was added in 2.6.16.
  • echo 1:释放页面缓存
    echo 2:释放目录文件和inodes
    echo 3:释放所有缓存(页面缓存，目录文件和inodes)*

zabbix在window下安装

发表于 2019-09-18 分类于 zabbix

1.下载好相应的win下的zabbix客户端包
2.解压到某个目录(目录任意，我一般都放在c盘根目录)
3.cmd下进入到zabbix的bin下的win64目录(我的目录是：C:\zabbix_agents_3.0.4.win\bin\win64,如果你安装32位的就进入到win32目录)
4.执行命令：zabbix_agentd -h

5.首先我们要添加到服务，然后再启动(添加到服务方便以后开户自动启动)
在这一步，我们要注意应该如何添加服务，不能直接执行命令：zabbix_agentd.exe -i，虽然这一步不会出问题，但到下一步启动的时候会报错误，还是乱码，情况如下图

经过修改cmd的默认编码得知,错误提示为：服务没有及时响应或控制请求，如下图：

其实就是没找到配置文件而已，所以我们添加服务的时候，应该指定配置文件的路径，用-c参数
zabb-x_agentd.exe -i -c C:\zabbix_agents_3.0.4.win\conf\zabbix_agentd.win.conf
这样再启动，就成功了

再看下任务管理器里，已经有zabbix的进程了

服务也已经启动了

3.再执行命令就可以看到正常的中文提示了

Device eth0 does not seem to be present

发表于 2019-09-18 分类于 vmware

####故障说明：
由于换电脑了，虚拟机不想再装一遍，就将整个目录都拷贝到另一个电脑，在新机器上启动时便提示下面错误：

####故障原因：
这是因为换电脑了，vmare workstation 就会重新生成MAC地址，这样vmare软件和虚拟机系统里配置的mac地址不一样，这就造成了上述错误。

####解决方法

#####第一种：
这种方法简单粗暴，直接删除/etc/udev/rules.d/70-persistent-net.rules，然后重启

####第二种：
1.修改/etc/udev/rules.d/70-persistent-net.rules，将里面eth0网卡的ip地址修改成vware的新地址
2.修改/etc/sysconfig/network-scripts/ifcfg-eth0 里的mac地址也替换了
3.重启网络

excel打开第二个表格很慢

发表于 2019-09-18 分类于 windows

系统信息：

系统版本：win10 企业版 1607
office版本：office 专业增强版 2016

问题：

excel打开第二个表格很慢，等很久也打不开(2013版本的office也是如此)

原因：

当前文件处于编辑状态下（即光标在单元格内）是不能打开新档的，打开新文档，直接转到当前打开的文档，如果不动，就会一直处于这个状态(我们的感觉就是卡住了)，这时鼠标随便在其它单元格点下，新文件就立马打开了(因为在点的过程中，已经退出了单元格的编辑状态)。所以当我们想打开第二个文件时，要做的就是使单元格不处于编辑状态
